Contraintes

Configuration requise

OmegaT peut être exécuté sur tout système intégrant l'environnement d'exécution Java (JRE, Java Runtime Environment). Le JRE étant désormais fourni avec OmegaT, il est inutile de le télécharger séparément. L'installation d'OmegaT a été concluante sur toutes les plateformes Windows (à partir de Windows 98), Macintosh OS X et Linux.

Les fichiers au format Microsoft Office XML peuvent être traduits directement dans OmegaT. Par contre, les fichiers créés dans un format MS Word, Excel ou Powerpoint plus ancien doivent être convertis dans les formats correspondants d'OpenOffice.org, Apache OpenOffice ou LibreOffice (suites bureautiques libres et gratuites). Vous pouvez obtenir Apache OpenOffice ou LibreOffice à partir de la page « Logiciels tiers » accessible via le menu sur la gauche de l'écran.

Profil utilisateur

Bien que l'installation et l'utilisation d'OmegaT ne présentent pas de difficulté particulière, il est vivement recommandé de prendre connaissance de la documentation fournie et de suivre les instructions indiquées. Si, d'une manière générale, vous avez une certaine aversion pour la lecture d'instructions et que vous avez tendance à découvrir un logiciel en cliquant partout dans l'attente que quelque chose ressemblant à l'effet recherché se produise, alors OmegaT n'est probablement pas le bon outil pour vous.

Pour la plupart des utilisateurs, elles sont sans intérêt Certaines restrictions imposées par le système d'exploitation ou la version Java utilisés pourront toutefois obliger certains utilisateurs à préférer une version plus ancienne.

Avec ou sans JRE

L'environnement d'exécution Java (JRE) doit être installé sur votre ordinateur pour qu'OmegaT puisse fonctionner.

Si un JRE adéquat est déjà installé sur votre ordinateur, vous pouvez télécharger une version d'OmegaT sans JRE – le téléchargement sera plus rapide et l'installation demandera moins d'espace sur le disque dur. Dans ce cas, choisissez par exemple une version « standard » ou « récente » puis sélectionnez la variante portant la mention correspondante (« Java not included », « without_JRE »).

Si vous deviez installer OmegaT avec son JRE, alors qu'un JRE est déjà présent sur votre ordinateur, il n'y aura pas de conflit entre les versions en raison d'installations distinctes. Par conséquent, l'installation d'OmegaT avec le JRE reste la méthode la plus sûre. Si vous n'êtes pas certain d'avoir une version appropriée du JRE, sélectionnez la version d'OmegaT avec JRE (ou la version par défaut).

Note à l'attention des utilisateurs de Mac OS X : le JRE étant installé par défaut sur Mac OS X, la version correspondante d'OmegaT ne comporte pas de JRE.

Note à l'attention des utilisateurs de Linux : la plupart des distributions Linux sont fournies avec un environnement d'exécution Java, cependant certaines implémentations Open Source ne fonctionnent pas toujours correctement avec OmegaT (incompatibilité qui ne s'avère éventuellement pas immédiatement). Nous recommandons aux débutants sur Linux de choisir une version d'OmegaT avec JRE. Les utilisateurs avertis pourront préférer une installation du JRE d'Oracle (anciennement Sun) au niveau système.

Signé / Non signé

Mac OS X dispose d'une fonctionnalité appelée « Gatekeeper » qui renforce la sécurité en contraignant l'exécution de programmes non signés. OmegaT pour Mac OS X est disponible en version signée et non signée ; elles sont fonctionnellement identiques, mais la version signée est préférée car elle est reconnue par le système comme étant publiée par un développeur connu.

Il peut y avoir un délai dans la publication des versions signées. Les utilisateurs pressés peuvent utiliser la version non signée ; si vous rencontrez des difficultés à lancer l'application, consultez l'article Comment ouvrir une application d'un développeur non identifié.

La version Web Start est une solution d'installation simple permettant d'exécuter la version actuelle d'OmegaT.

L'une des particularités de cette version est l'installation automatique des mises à jours.

Durant l'installation, en fonction de votre système d'exploitation, vous recevrez peut-être plusieurs avertissements de sécurité.
Le certificat est signé par Concept PnS Sarl, une société française.

Les autorisations que vous accordez à cette version (par exemple, un « accès illimité à l'ordinateur ») sont identiques aux autorisations que vous donnez à la version locale (c'est-à-dire un accès au disque dur de l'ordinateur).

L'installation de la version Web Start requiert Java 1.3 ou ultérieur sur votre ordinateur.

Engagement de confidentialité : le Java Web Start d'OmegaT ne stocke aucune de vos données sur les serveurs Web d'OmegaT. L'application est exécutée sur votre machine ; vos documents et mémoires de traduction restent sur votre ordinateur et le projet OmegaT n'aura en aucun cas accès à vos travaux ou informations.

La version de développement (alpha) reflète la phase de développement la plus récente. Elle n'est recommandée qu'aux développeurs et testeurs assidus. Elle peut également être obtenue via la commande de Subversion :

svn validation svn://svn.code.sf.net/p/omegat/svn/trunk omegat-svn

Des compilations périodiques sont disponibles sur CloudBees.

Le code source n'est pas nécessaire à l'utilisation du logiciel, il n'est destiné qu'aux développeurs. Ils sont placés dans les dossiers des versions compilées correspondantes.
