Remarques générales

L'équipe du projet OmegaT The OmegaT project team is not a company or formal legal body, just a group of volunteers involved in the OmegaT project.

OmegaT est un logiciel libre et open source de TAO dont l'interface utilisateur et la documentation ont été écrites en anglais. Étant destiné à un public international du domaine des métiers de la traduction, il est souhaitable de le mettre à la disposition du plus grand nombre possible et donc de le localiser dans d'autres langues. However, since we are not a commercial project, we have no budget for the commissioning of translations. Aussi, les contributeurs du projet OmegaT espèrent-ils un retour de participation de la part de professionnels qui auront fait leurs expériences avec le logiciel et seront convaincus de son mérite pour la communauté des traducteurs dans son ensemble.

Traduction

Actuellement, le contenu à traduire comprend les parties suivantes :

* The user interface (menus etc.)
* The user manual (very comprehensive)
* The Instant Start guide (a short introduction to using OmegaT)
* The readme file (a document that contains general information about OmegaT)
* The Beginner's Tutorial
* le site Web, avec le cas échéant les guides pratiques disponibles en téléchargement.

Les traducteurs bénévoles peuvent s'investir et apporter leur contribution en fonction de leurs compétences et du temps dont ils disposent – In particular, the main user manual is likely to take around two weeks to translate, and several translators may wish to share the work.

Procédure

1. Contact the relevant co-ordinator for the translation work:
For the user interface, user manual, Instant Start guide and/or the readme file, please contact the OmegaT Localization Co-ordinator.
If you wish to translate the OmegaT website and/or the Beginner's Tutorial, please contact the OmegaT Website Manager.

2. Inscription : rejoignez le groupe des utilisateurs d'OmegaT (ou « OmT »). Durant le processus d'abonnement, vous serez invité à envoyer un bref message aux modérateurs du groupe : ceci leur permet de distinguer les vrais utilisateurs des spammeurs.

Une fois votre inscription validée, consultez le plan de localisation d'OmegaT. Ce plan détaille l'avancement du travail de localisation : il répertorie les tâches en cours de traduction ainsi que les travaux déjà achevés. Si l'une des parties mentionnées plus haut n'a pas encore été traduite, vous pouvez la réserver en ajoutant une entrée indiquant que vous prenez la responsabilité de sa traduction. Le but de ce plan est d'éviter toute répétition inutile du travail si un autre traducteur se consacre déjà à la traduction en question. La réservation d'une tâche est une étape très importante : veillez à ajouter votre entrée dans le plan de localisation correspondant avant de vous lancer dans une traduction.

3. L'abonnement à la liste de diffusion des localisateurs d'OmegaT (« OmT-l10n ») est également vivement recommandé, même si ceci reste secondaire.

4. Download the files for translation.

For the OmegaT program and documentation, these are either the "Full" package or the "Minimal" package.

Le paquet « Full » contient le manuel de l'utilisateur, le tutoriel de prise en main, les ressources de l'interface utilisateur et le fichier lisez-moi. Le contenu du paquet « Minimal » est identique, à l'exception du manuel de l'utilisateur. Vous n'êtes pas obligé de traduire la totalité d'un paquet ; vous pouvez par exemple vous limiter à l'interface utilisateur (Bundle.properties) ou au tutoriel de prise en main.

If you wish to translate the OmegaT web pages, download the website localization package. Remarque : il s'agit des fichiers rédigés en anglais. If you prefer to translate from a different language into which OmegaT has already been localized, please contact the OmegaT webmaster.)

Localizations of the OmegaT user interface and documentation are generally updated in step with new releases of OmegaT.

.

Les fichiers du site Web d'OmegaT sont aux formats HTML et, depuis 2011, PHP. To translate the latter in OmegaT, you must register the .php extension with the HTML and XHTML filter as follows:

Options > File Filters > Select "HTML and XHTML files" > Edit > Add > Then enter "*.php" in the "Source Filename Pattern" column.

Finally, there is the Beginner's Tutorial, available here.

5. Certaines références (anciennes versions de la documentation, mémoires de traduction sous forme de fichiers TMX, etc.) existent éventuellement déjà et peuvent par conséquent s'avérer utiles pour votre travail. Translation memory (TMX) files of older localizations of OmegaT, the documentation and the website can be found in the "Files" section of the online user group. It is also worth asking on the localizers' list (OmT-l10n) whether anyone has any reference material which might facilitate your translation.

6. Traduction des fichiers : bien entendu, nous vous encourageons à travailler avec OmegaT, quoique vous soyez libre d'utiliser le logiciel qui vous convient.

After downloading the zip archive of the full or minimal package or the website localization package, unpack the archive. For the full and minimal packages (user interface and documentation), you do not need to create an OmegaT project as you normally would in OmegaT: when unpacked, the files for localization of the OmegaT program are a ready-to-use OmegaT project. (This is not the case for the website localization files.)

Votre traduction n'est soumise à aucune contrainte particulière – vous veillerez toutefois à observer l'utilisation d'un langage correct et à transmettre des informations pertinentes que vous pouvez choisir de modifier, d'ajouter ou d'ignorer. If you have any questions, the OmT-l10n list is the best place to post them.

Si le document que vous traduisez contient la GPL (GNU Public License) ou la GFDL (GNU Free Documentation License), veuillez la conserver dans sa version originale, ne la traduisez pas – d'une part car la version originale en anglais est la seule version juridiquement valable (certes, une traduction sans valeur juridique peut être fournie à titre d'information), Secondly, unless yours is a very uncommon language, both documents have in all probability already been translated. (si vous faites une recherche sur le Web, vous trouverez certainement les traductions correspondantes).

Please keep translations of files in their original format, and do not change the file names. si vous travaillez avec OmegaT, le logiciel conservera systématiquement le formatage. Nous sommes très reconnaissants de toutes les contributions qui nous parviennent, mais vous nous faciliterez le travail en nous remettant les fichiers dans leur format d'origine (et non en format Word, par exemple).

7. Tests de localisation (facultatifs)

Test de l'interface utilisateur :

Lancez OmegaT à partir de la ligne de commande (fenêtre MS-DOS) avec les commandes suivantes (si vous avez des doutes, consultez le manuel de l'utilisateur d'OmegaT) :

     cd [full path to OmegaT]
     java -jar OmegaT.jar resource-bundle=[name of bundle with full path]

[full path to OmegaT] étant le chemin complet du dossier du programme OmegaT et [name of bundle with full path] le nom de votre traduction pour le paquet de ressources correspondant (par exemple, bundle_en.properties) avec le chemin complet vers son emplacement.

Test d'affichage du tutoriel de prise en main au lancement du logiciel :

Dans le dossier /docs/ du programme OmegaT, créez un sous-dossier et attribuez-lui comme nom le code ISO de la langue de votre système d'exploitation, puis copiez-y le fichier du tutoriel de prise en main (si ce sous-dossier existe déjà, renommez-le temporairement). Procédez de la même manière pour relire le manuel de l'utilisateur dans OmegaT. Lancez OmegaT comme vous y êtes habitué. Si vous avez traduit dans une langue qui n'est pas la langue de votre système d'exploitation, référez-vous au point « Gestion des versions localisées par OmegaT ».

8. Produce a screenshot of OmegaT in your language (after translating the user interface). cette étape est également facultative, mais si vous êtes en mesure de lancer OmegaT avec votre localisation (voir ci-dessus), elle apportera un plus à la qualité de votre traduction. Lancez OmegaT et organisez un écran approprié, par exemple l'environnement d'édition avec la boîte de dialogue des propriétés d'un (nouveau) projet affichant les chemins des différents dossiers.

Enregistrez votre capture d'écran avec le type de fichier PNG.

N'oubliez pas d'attribuer un nom explicite à l'image (par ex. NouvelleImage.png).

9. Deliver your file to the localization co-ordinator, who will pass it on to the development team for inclusion in the next release of OmegaT (user interface and documentation) or upload it to the website (website pages). If you used OmegaT or another CAT tool for the work, please also upload your translation memory (TMX file) to the translation memory repository in the online user group. Ceci facilitera les mises à jour de votre traduction pour les versions futures. Veuillez noter que le coordinateur de la localisation n'a en aucun cas besoin de la totalité votre projet OmegaT : préférez la création de fichiers zip ou tar.gz, l'un avec les documents cibles, l'autre avec les fichiers TMX.

10. Don't forget to update the OmegaT Localization Plan (see Step 2 of the procedure) to show that you have completed translation of a particular document/version. ou de son abandon si vous deviez décider le l'interrompre. Ceci permettra de libérer le document en question pour qu'un autre traducteur puisse prendre la relève. If you discontinue before completing the translation, please do create the tmx files for the you have done and upload them to the translation memory repository.
