OmegaT peut être installé sur tout système d’exploitation intégrant la version 11 de l’environnement d’exécution Java.

Choisissez l’une des deux options de téléchargement proposées : avec ou sans environnement d’exécution Java (JRE). Si un JRE adéquat est déjà installé sur votre ordinateur, sélectionnez « Sans JRE ». Si votre système ne dispose pas de la version appropriée, ou si vous n’en êtes pas certain, sélectionnez « Avec JRE ».

OmegaT peut normalement être exécuté sur tout système Linux intégrant l’environnement d’exécution Java (JRE). OmegaT requiert actuellement le JRE version 11 ou ultérieure. Pour vérifier la version présente sur votre système Linux, ouvrez une console (fenêtre de ligne de commande) et saisissez :

$ java -version

Si vous n’êtes pas certain d’avoir une version appropriée du JRE, vous pouvez télécharger et installer une version d’OmegaT avec JRE. Cette méthode est sûre et exclut tout conflit de versions (risques de remplacement/désinstallation) puisque ce JRE sera utilisé uniquement localement pour OmegaT.

Version d’OmegaT « piles incluses », fournie avec son environnement d’exécution Java. Son téléchargement est plus long et son installation demande plus d’espace sur le disque dur, mais elle est certaine de fonctionner. En cas de doute, choisissez cette option.

Le JRE inclus est la version 64 bits.

Versions obsolètes

Vous pouvez télécharger d’anciennes versions obsolètes ​​d’OmegaT.

Ces versions obsolètes sont des versions antérieures qui ont été remplacées. Pour la plupart d’entre vous, elles sont sans intérêt. Certaines restrictions imposées par le système d’exploitation ou la version Java utilisés pourront toutefois obliger certaines personnes à préférer une version plus ancienne.

Avec ou sans JRE

L’environnement d’exécution Java (JRE) doit être installé sur votre ordinateur pour qu’OmegaT puisse fonctionner.

Si un JRE adéquat est déjà installé sur votre ordinateur, vous pouvez télécharger une version d’OmegaT sans JRE – le téléchargement sera plus rapide et l’installation demandera moins d’espace sur le disque dur. Dans ce cas, choisissez par exemple une version « standard » ou « récente » puis sélectionnez la variante portant la mention correspondante (« Java not included », « without_JRE »).

Si vous deviez installer OmegaT avec son JRE, alors qu’un JRE est déjà présent sur votre ordinateur, il n’y aura pas de conflit entre les versions en raison d’installations distinctes. Par conséquent, l’installation d’OmegaT avec le JRE reste la méthode la plus sûre. Si vous n’êtes pas certain d’avoir une version appropriée du JRE, sélectionnez la version d’OmegaT avec JRE (ou la version par défaut).

Si vous utilisez Linux : la plupart des distributions Linux sont fournies avec un environnement d’exécution Java. Cependant, certaines implémentations en source libre ne fonctionnent pas toujours correctement avec OmegaT (incompatibilité qui ne s’avère pas toujours immédiatement). Nous recommandons aux débutants sur Linux de choisir une version d’OmegaT avec JRE. Les personnes averties pourront préférer une installation du JRE d’Oracle (anciennement Sun) au niveau système.

Notarié et non signé

macOS tente d’augmenter la sécurité en rendant plus dure l’exécution de programmes non vérifiés.

OmegaT pour macOS est disponible en version notariée et en version non signée ; les deux versions sont fonctionnellement identiques, mais la version notariée est préférée, car elle est reconnue par le système comme étant a) publiée par une personne enregistrée et b) scannée par Apple.

Il peut y avoir un délai dans la publication des versions notariées. Les personnes pressées peuvent utiliser la version non signée ; si vous rencontrez des difficultés à lancer l’application, consultez l’article Comment ouvrir une application d’une personne non identifiée.

La version de développement (alpha) reflète la phase de développement la plus récente. Elle n’est recommandée que pour tester les nouvelles fonctions d’OmegaT. Elle peut également être obtenue via la commande git suivante :

git clone https://git.code.sf.net/p/omegat/code omegat-code

Des compilations périodiques sont disponibles sur SourceForge.

Les paquets contenant le code source ne sont pas nécessaires à l’utilisation du logiciel, ils ne sont destinés qu’au développement. Ils sont placés dans les dossiers des versions compilées correspondantes.
